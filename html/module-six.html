6. Introduction to Version Control with Git

Version control is a system that records changes to a file or set of files over time so that you can view, restore, or compare different versions. Git is a popular version control system that is commonly used for software development.

6.1 What is Version control

Version control is a system that records changes to a file or set of files over time, allowing you to view, restore, or compare different versions. It helps keep track of changes made to a project, make it easier to collaborate with others, and maintain a clean and organized codebase.

6.2 Installing Git

Git is a powerful and widely-used distributed version control system that allows developers to track changes, collaborate on projects, and manage code efficiently. It does not require a central server to function, making it versatile for both personal and collaborative projects.

Installing Git is the first step to start managing your projects with version control. Depending on your operating system, there are different ways to install Git. Below is a step-by-step guide to help you set it up:

Installing Git on Windows

  • Visit the official Git website: Git Downloads.
  • Download the Git for Windows installer.
  • Run the installer and follow the setup wizard.
  • Select recommended default settings during installation.
  • Verify the installation by typing git --version in Command Prompt.

Installing Git on macOS

  • Open the Terminal application.
  • Type git --version to check if Git is already installed.
  • If not installed, download Git from Git Downloads or use Homebrew:
  • brew install git (requires Homebrew installed).
  • Confirm installation with git --version.

Installing Git on Linux

  • Open your terminal.
  • For Debian-based systems (Ubuntu): sudo apt-get install git
  • For Red Hat-based systems (Fedora): sudo dnf install git
  • Verify installation with git --version.
  • Configure your Git username and email:
    • git config --global user.name "Your Name"
    • git config --global user.email "you@example.com"

After installing Git, it's recommended to set up your username and email, as shown in the Linux section, to personalize your commits. By following these steps, you can successfully install Git and start managing your code efficiently.

6.3 Git Basics

Git is a powerful command-line tool that allows developers to manage their code versions efficiently. Knowing the fundamental Git commands is essential for version control and collaboration. Below are some basic Git commands to get you started:

Initializing a Repository

Use git init to create a new Git repository in your project folder. This command sets up a hidden .git directory, enabling version control.

Checking Status

The git status command shows the current status of your working directory and staging area. It helps track changes and identify staged files.

Adding Files

Use git add to stage files for committing. You can add specific files or use . (dot) to stage all changes.

Committing Changes

Use git commit -m "Your message" to create a commit with a message describing the changes. Commits are saved snapshots of your project.

Viewing Commit History

The git log command displays the commit history, showing commit IDs, authors, dates, and messages.

6.4 Git Collaboration & Workflows

Git collaboration is a powerful feature that allows developers to work together on a project by using branches, merging changes, and resolving conflicts. Here are some Git collaboration workflows:

Creating Branches

Use git branch to create a new branch with the current commit. Branches help you isolate features or bug fixes.

Switching Branches

The git checkout command lets you switch between branches, allowing you to work on different versions of your project.

Merging Branches

Use git merge to combine changes from one branch into another. It's essential for integrating features.

Rebasing Branches

The git rebase command reapplies commits from one branch onto another, helping keep a linear commit history.

Stashing Changes

Use git stash to temporarily save uncommitted changes, so you can work on something else without losing progress.

6.5 Git Branching & Merging

Git branching and merging are essential Git workflows for collaborating on a project. Here are some tips for managing branches and merging changes:

Step 1: Create a Branch

Create a separate branch for each feature or bug fix you work on. This helps isolate changes and prevents conflicts.

Step 2: Make Changes & Commit

Implement your changes and use git commit to save them to your branch.

Step 3: Create a Pull Request

When ready, open a pull request to merge your branch into the main branch. This facilitates code review.

Step 4: Review & Approve

Collaboratively review the pull request with your team, making sure the code meets standards before merging.

Step 5: Delete Merged Branch

After successfully merging, delete the branch to keep the repository clean and organized.

Conclusion

Git is a powerful tool for version control and collaboration in software development. By understanding its basics, using branching and merging workflows, and staying up-to-date with the latest features and bug fixes, you can become more efficient and effective in your project. Good luck with your learning journey!
